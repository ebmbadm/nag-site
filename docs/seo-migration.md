# SEO — миграция Tilda → Next.js (novikamps.com)

Аудит и работы: 2026-06-30. Источник данных: Яндекс.Метрика (счётчик 94760065),
Яндекс.Вебмастер (хост `https:novikamps.com:443`).

## Контекст

Текущий live-сайт — **Tilda**. Этот Next.js-репозиторий заменяет его на **том же
домене** novikamps.com. Прод — **Timeweb (Docker, `next start`)**, поэтому
server-side 301 и оптимизация картинок работают. Vercel — только тестовый полигон.

Поиск: **Yandex ~58% / Google ~41%** органики (≈59% всего трафика). ИКС = 30.
Запросы — в основном брендовые/модельные (novik, усилитель nag, ламповый усилитель
novik 1202, nag d8000, nag tds-20). Многие позиции 1–3 — их и защищаем редиректами.

## Чеклист выката (cutover) — выполнить при переключении домена на новый сайт

1. **До переключения DNS/хостинга:** прогнать прод-сборку (`npm run build && npm start`),
   проверить 301 на ключевых старых URL (`/d8000`→`/catalog/d-8000`, `/qm400`,
   `/history`→`/istoriya`, `/savers/tproduct/*`→`/catalog/savers`). Главная `/` = 200.
2. **robots.txt** доступен на `https://novikamps.com/robots.txt`, **sitemap.xml** на
   `https://novikamps.com/sitemap.xml` (отдаются автоматически Next-роутами).
3. **Яндекс.Вебмастер:** Индексирование → Файлы Sitemap → добавить
   `https://novikamps.com/sitemap.xml`. Переобход важных страниц (новые `/catalog/*`,
   `/istoriya`, `/kontakty`). Проверить «Страницы в поиске» на рост 404 — старые слаги
   должны отдавать 301, а не 404.
4. **Google Search Console:** Sitemaps → добавить sitemap. URL Inspection → запросить
   индексацию для главной и топ-категорий.
5. **Зеркало/протокол:** убедиться, что `http→https` и `www→без www` тоже 301 (уровень
   хостинга/Timeweb, вне Next).
6. **Мониторинг 2–4 недели:** в Вебмастере — индексируемые страницы, запросы/позиции;
   в Метрике — органика по посадочным. Любой старый URL, отдающий 404, добавить в
   `lib/redirects.ts`.

## Открытые вопросы (нужно подтверждение)

1. **`/f8000` и `/f8wifi` → `/catalog/f-8`.** Оба слага про Wi-Fi; страница `f-8` —
   именно Wi-Fi/FIR (у `f-8-pro` — AES, без Wi-Fi). Если исторически «F8000» = флагман,
   поменять `/f8000` → `/catalog/f-8-pro`.
2. **`/bt` (16 визитов) → `/catalog/f-8`.** Bluetooth-страницы/товара нет; F-8 —
   ближайший беспроводной DSP. Альтернатива — `/catalog/processors`.
3. **Низкоприоритетные Tilda-id** `/c5`–`/c43`, `/gruppa-5/7`, легаси `/page*.html`
   (1–10 визитов) — целевые категории угаданы. Если есть старая Tilda-карта сайта,
   можно уточнить точно.
